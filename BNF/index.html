このページで用いられている記法について

Extensible Markup Language (XML) 1.0 (Fifth Edition)に準ずる表記法です。
BNFを拡張した表記法ですが、EBNF や ABNFではありません。
上記リンク先の記法と異なる点が4つあります。

  1. 終端記号をクオテーションで括っていない。
    色付けしているので混乱はしないと思います。例："int"ではなく、int
  2. 終端記号としての空白類文字を明記していない。
    以下のルールで記述しています。
    • 定義中にスペースがない箇所 → スペース挿入不可。
    • 定義中にスペースがある箇所で、前後ともに[0-9A-Za-z_] → スペース必須。
    • その他 → スペースの存在は任意。
    • Import構文( Import ) や 文( Sentence )の終わりには改行文字\n(LF)が必要。[要出典]
    • その他改行文字が必要な箇所ではBNFの定義中でも改行されている。逆に行の途中では改行不可能。[要出典]
  3. 定義の終了を 』 で示している。
    セミコロン( ; )で示す流儀もありますが、 』 の方が誤認識しにくいと考えました。
  4. 1行コメントを採用。
    例：// コメント

BNF

KuinSource ::= ( Import | Definition )* 』
// Kuinのソースファイルの拡張子は.knです。
// Kuinのソースファイルは UTF-8 で書いてください（Ascii でもいけます）。 [出典: Twitter / kuina_tesso]

import構文

Import ::= import SourceName 』
// Test.knをインポートする場合の例: import Test
// Test.knに定義された関数、変数、クラスなどが扱えるようになります。
// 例えば、Test.kn上の F() 関数を呼び出すときは、Test@F() とします。ソースコード名が、そのまま名前空間となります。
// Kuin では、誰が書いても同じようなコードになるという設計上、自身のソースコード上のグローバル変数であっても、@ の記述を必要とします。
// つまり、Test.kn 内部から Test.kn 内部のグローバル変数やグローバル関数を参照するときも、Test@F() などと書く必要があります。
// Test@ が省略できるのは、ローカル変数やローカル関数などへの参照だけです。
// 
// 使われていない関数や変数などは、コンパイル時に勝手に外されます。
// なので、使われないソースコードを無駄にimportしたからと言って、実行時の効率が悪くなることはありません。

文

Sentences ::= Sentence+ 』
Sentence  ::= Definition     |  // 定義文
              SimpleSentence |  // 単文
              BlockSentence  』 // ブロック文

定義文

Definition ::= -? (Func | Var | Const | Alias | Class | Enum )』
// - ⇒ 他のファイルに対してprivateな宣言[要出典]。 Kuin 0.023ではコンパイルエラーにはならないけれど未実装。

func構文 (関数定義)

Func ::= func FName ( (VName : &? Type( , VName : &? Type )*)? )( : Type)?
             Sentences?
         end func 』
// 関数の引数の型の前に & を付け、関数呼び出し時( FunctionCall )も & を付けると、参照渡しになります。
// 例：func f(x : &int) に対し、do f(&i) [出典: Twitter / kuina_tesso]

var構文 (変数定義)

Var ::= var VName : Type( :: Expr)? 』
// 代入( :: Expr)を省略した場合、すべて0で初期化されます。 [出典: Twitter / kuina_tesso]
// グローバル変数・メンバ変数では、定義時には値を代入できません。
// 定義と同時に値を代入できるのは、ローカル変数のみです。

const構文 (定数定義)

Const ::= const VName : Type :: Expr 』
// 定数の代入式の右辺(Expr)は、コンパイル時に決定できる値でなくてはなりません。

alias構文 (別名定義)

Alias ::= alias AName : Type 』
// []char を string などの別名にするのは推奨されません。
// ( Kuin の文字列型が []char であることに慣れるのが望ましい )

class構文 (クラス定義)

Class ::= class CName ( ClassName? )
              ( [+-]? *? Definition )*
          end class 』
// 継承元( ClassName )を省略した場合、Kuin@CClass が継承されます。
// 全てのクラスは、ルートクラスである Kuin@CClass が継承されていると言えます。
// 例： class CCat : CAnimalとした場合、Kuin@CClass → CAnimal → CCat
// 
// - ⇒ private なメンバ。
// + ⇒ protected なメンバ。
// * ⇒ 親クラスのメンバのオーバーライドを許可。
// * は、- や + の後に記述しなければエラーとなります。
// (誰が書いても同じようなコードにするため)
// ※Kuin 0.022 段階では、private / protected 指定は無視されます。[出典: Twitter / 10000favs]
// オーバーライドできるのは関数のみです。[出典：Error: E0033]
// 
// コンストラクタは、*func Ctor(), デストラクタは*func Dtor()[詳細: Twitter / 10000favs]
// {インスタンスを作って初期化(Init)する方法} [出典: Twitter / yuhi_as]
// {クラスのInitメソッドの宣言} 
// func Init(仮引数) : くらす
//     do this.メンバ変数 :: 引数
//     return this
// end func
// {呼び方}
// var c : くらす :: (@new くらす).Init(実引数)
// 上記のように、thisが使えます。 [出典: Twitter / yuhi_as]

enum構文 (列挙体定義)

Enum ::= enum EName
             ( ConstName( :: Expr)? )*
         end enum 』
// :: Exprを指定しなかったときのデフォルト値は 0 から始まります。
// 値は自動的に 1 ずつ足されて設定されます。
// 下記の例では、Red は 0、Blue は 1、Yellow は 6に設定されます。
// enum EColor
//     Red
//     Blue
//     Green :: 5
//     Yellow
// end enum
// 他の要素と値が重複した場合は、コンパイルエラーとなります。
// 値の指定は、コンパイル時に int に決定される定数でなくてはなりません。

ブロック文

BlockStatement ::= If | Switch | While | For | Foreach | Try | Ifdef | Block 』
// ブロック内で定義したローカル変数は、そのブロック内でのみ参照できます。

if文

If ::= if BlockName? ( Expr )
           Sentences?
       ( elif ( Expr )
           Sentences? )*
       ( else
           Sentences? )?
       end if 』
// 条件式 (()の中身)は、bool型でなければなりません。
// C言語のように int型にした場合、コンパイルエラーとなります。

switch文

Switch ::= switch BlockName? ( Expr )
           ( case ( Value )
               Sentences? )*
           ( default
               Sentences? )?
           end switch 』
// フォールスルーできません。case の最後に達した段階で自動でブロックを抜けます。
// case の値は、カンマ区切りで複数指定できます。また、@to 演算子により、範囲指定もできます。
// 上記のExprの型とValueの定義中のExprの型は一致しなければなりません。
// Exprに指定できる型： int、byte(byte16など)、char、enum、[]char
// case の値は、コンパイル時に定数とならなくても構いません。
// 複数の case条件に合致するとき、最初(最も上)に書かれた caseに捕捉されます。
// case節で()が必要になったのは、Kuin 0.021以降です。[出典: Twitter / 10000favs]

while文

While ::= while BlockName? ( Expr( , skip)? )
              Sentences?
          end while 』
// 条件式は bool型でなくてはなりません。
// skipを指定した場合、初回の条件比較をスキップします (C言語のdo-whileの代替)。

for文

For ::= for BlockName? ( Expr , Expr( , Expr)? )
            Sentences?
        end for 』
// 括弧内は順に、初期値、終値、増減値です。
// 増減値を省略した場合、増減値は1になります。
// 増減値が正の数ならば i <= 終値 がループ続行の条件となります。
// 増減値が負の数ならば i >= 終値 がループ続行の条件となります。
// 増減値が0の場合はコンパイルエラーになります。
// 増減値はコンパイル時に定数になる値でなければなりません。

foreach文

Foreach ::= foreach BlockName? ( Expr )
                Sentences?
            end foreach 』
// foreach で使えるのは、配列(文字列含む)、list、stack、queue、dict型だけです。
// dict型からforeachで要素を取り出すと、KeyとValueのペアを持ったdictpair型になります。
// enumも使えるようになるかも？ [出典: Twitter / 10000favs]

try文

Try ::= try BlockName?
            Sentences?
        catch ( Value? )
            Sentences?
        ( finally
            Sentences? )?
        end try 』
// 上記のValueで例外コード(int型)を指定します（複数指定可能）。
// catchで指定する例外コードは、コンパイル時に定数とならなくてはなりません。
// 例外の情報は、ブロック名(ここではeとします)を用いて、e.Code、e.Msg のように参照できます。
// eはKuin@CExcptクラスに属しており、例外が発生したソースコード上の箇所がこの変数に格納されています。
// e.Code は例外コード(int型)、e.Msg はメッセージ([]char型)です。
// tryやcatch中でbreakした場合、finallyを通ります。 [出典: Twitter / 10000favs]
// ブロック名直後の括弧内に発生を抑制したい例外コードを記述する機能は削除されました。
// [出典: Twitter / 10000favs]
// catch節に()が必要になったのは、Kuin 0.02以降です。 [出典: Twitter / 10000favs]
// finally内で そのtry文をbreakすると、無限ループになってしまうので注意してください。

ifdef文

Ifdef ::= ifdef BlockName? ( (release|debug) )
              Sentences?
          end ifdef 』
// debug を指定するとデバッグコンパイル時のみ中身がコンパイルされます。
// C言語のプリプロセッサである #ifdef に似ていますが、ブロックである点に注意です。
// つまり、Kuin の ifdef文の中で宣言した変数等は、その中でのみ使用できます。
// 
// デバッグ用関数内で ifdef して リリース時に中身が空としてコンパイルされることを想定。
// リリース時は最適化が働き、空の関数は存在が消えるなどの工夫を施されるそうです。[出典: @10000favs]([出典: @10000favs] [出典: @10000favs] )

block文

Block ::= block BlockName?
              Sentences?
          end block 』
// 単純に、ブロック構造を作りたいときに有効です。
// 他のブロック文と同様、breakできます。

単文

SimpleSentence ::= Do | Break | Continue | Return | Assert | Throw 』

do文

Do ::= do Variable OpSubstitution Expr | // 代入
       do Variable OpSwap Variable | // 値の交換
       do FunctionCall 』 // 関数呼び出し
// 代入文では、両辺が参照型の場合、値ではなくアドレスが代入されます。

break文

Break ::= break BlockName 』
// BlockNameで指定したブロックを抜けます。
// BlockNameは省略不可能です。 [出典: Twitter / kuina_tesso]
// ただし、breakしないブロックにはBlockNameは不要です。[出典: 言語仕様Ver.0.03]
// 全てのブロック文でbreak可能です。

continue文

Continue ::= continue BlockName 』
// BlockNameという名前を持つブロックの終端(end)直前にジャンプします。
// BlockNameは省略不可能です。 [出典: Twitter / kuina_tesso]
// continue可能なブロックは、while、for、foreachです。

return文

Return ::= return Expr? 』
// 戻り値が無い関数では、値を省略して return のみを記述することになります。

assert文

Assert ::= assert Expr 』
// 上記のExprはbool型でなければなりません。
// 例： assert a >= 2
// このとき、デバッグ実行時に a が 2以上でなければ、例外が投げられます。
// a が 2以上であることが保証されるべき場面での、テストに用いられます。
// リリース時にはコンパイルから外されますので、最終的な動作速度に影響しません。

throw文

Throw ::= throw Expr( , Expr)? 』
// 1つ目のExprは例外コード( int型 )
// 2つ目のExprは例外メッセージ( []char型 )
// 例外メッセージを省略した場合、例外メッセージに null が入ります。

式・変数

Expr ::= ( Expr ) |
         String | Char | Boolean | Number |    // リテラル
         Variable | ConstantName | // 変数・定数
         BlockName |                   // (Forなどの)ブロック名
              Op1 Expr |               // 単項演算
         Expr Op2 Expr |               // 二項演算
              OpType1 Type |           // インスタンスの作成(@new)
         Expr OpType2 Type |           // キャスト演算($)
         Expr OpClass2 ClassName |     // クラスチェック(@is、@nis)
         Expr ?( Expr , Expr ) |       // 条件演算
         FunctionCall 』               // 関数呼び出し
// 条件演算の ? と ( の間にスペースを入れてはいけません。
// true ?(A, B)の場合はAのみ、false ?(A, B)の場合はBのみが評価されます。
// デバッグ実行時に配列の範囲外をアクセスすると、例外が投げられます。
// 例えば、s[5]まで定義されている配列があるとして、 s[-1] や s[6] は例外発生です。
// これにより、バグがメモリを破壊することを未然に防止できます。

Variable ::= VariableName |
             Variable [ Expr ] 』 // 配列アクセス

文字列リテラル

String ::= " ( [^"] | \" )* " |
           null 』

文字リテラル

Char  ::= ' ( [^'] | EscapedCh ) ' 』
// EscapedCh は \n \' などのエスケープ文字を表す文字列
// \0はKuinでは使用できません。 [出典: Twitter / 10000favs]

Booleanリテラル

Boolean ::= true | false 』

数値リテラル

// Kuinでは、16進数はもちろん、2～36進数の浮動小数点まで記述できます。
// 2進数の表記は例えば、2#00101111.1101 のようになります。#の前が基数です。
Number ::= [+-]?NonNegativeNumber(eNumber)? |
           inf』
// 指数表記 (31416e-4など)はKuin 0.02では未実装とのこと。 [出典: Twitter / kuina_tesso]
// infはfloat型のリテラルです[要出典]。
NonNegativeNumber ::= Base2 | Base3 | ... | Base36 』  // 2進数～36進数
Base2  ::=  2#[0-1]+(.[0-1]+)? 』
Base3  ::=  3#[0-2]+(.[0-2]+)? 』 
// Base4, Base5, ..., Base8も同様
Base9  ::=  9#[0-8]+(.[0-8]+)? 』
Base10 ::=    [0-9]+(.[0-9]+)? 』        // 10#99はコンパイルエラー
Base11 ::= 11#[0-9A-A]+(.[0-9A-A]+)? 』
Base12 ::= 12#[0-9A-B]+(.[0-9A-B]+)? 』
// Base13, Base14も同様
Base15 ::= 15#[0-9A-E]+(.[0-9A-E]+)? 』
Base16 ::=   #[0-9A-F]+(.[0-9A-F]+)? 』 // 16#FFはコンパイルエラー
// 16進数の基数は省略しなければなりません(誰が書いても同じになるように)。
Base17 ::= 17#[0-9A-G]+(.[0-9A-G]+)? 』
Base18 ::= 18#[0-9A-H]+(.[0-9A-H]+)? 』 
// Base19, Base20, ..., Base35も同様
Base36 ::= 36#[0-9A-Z]+(.[0-9A-Z]+)? 』

その他

識別子

Name ::= [A-Za-z_][0-9A-Za-z_]* 』
SourceName ::= Name 』 // ソースコード名
BlockName  ::= Name 』 // ブロック名
ClassName  ::= Name 』 // 定義時のクラス名
ConstName  ::= Name 』 // 定義時の定数名
AName      ::= Name 』 // 定義時の別名
FName      ::= Name 』 // 定義時の関数名
VName      ::= Name 』 // 定義時の変数名
EName      ::= Name 』 // 定義時の列挙体名
AliasName    ::= (SourceName@)?((ClassName.)*ClassName.)?AName 』 // アクセス時の別名
FunctionName ::= (SourceName@)?((ClassName.)*ClassName.)?FName 』 // アクセス時の関数名
VariableName ::= (SourceName@)?((ClassName.)*ClassName.)?VName 』 // アクセス時の変数名
EnumName     ::= (SourceName@)?((ClassName.)*ClassName#)?EName 』 // アクセス時の列挙体名
ConstantName ::= (SourceName@)?((ClassName.)*ClassName.)?ConstName |
                 EnumName#ConstName 』
// グローバル変数のアクセス時には「SourceName@」が必要です。
// enumの要素にアクセスする記法の例: Test@C#E#Hoge [出典: Twitter / kuina_tesso]
// (ソースTest 内のクラスC 内の列挙体E 内の要素Hogeにアクセス)

演算子

// 代入演算子が = ではなく、::なのが特徴的です。
// C言語の ==, !=, &&, || は、Kuinではそれぞれ、=, <>, &, | に対応します。
// &, | はC言語の&&, ||と同様に短絡評価される論理演算子です。
// インクリメント演算子はありません。i :+ 1 で代用してください。
Op1 ::= + | - | ! 』
Op2 ::= + | - | * | / | % | ^ | // 加減乗除、剰余、累乗
        @in | @nin | // 例: value @in array
        = | <> | < | > | <= | >= | // 等価、不等価、比較
        & | | | // 論理積、論理和
        ~ 』 // 配列連結
OpType1 ::= @new 』 // インスタンスの作成
OpType2 ::= $ 』 // キャスト演算子
OpClass2 ::= @is | @nis 』 // クラスをチェックする演算子
OpSubstitution ::= :: | // 代入演算子
                   :+ | :- | :* | :% | :^ | :~ 』 // 複合代入演算子
// 代入演算の戻り値はvoidです。[出典: Twitter / 10000favs]
OpSwap ::= @swap 』 // @swap演算子
// a @swap bで変数a と b の値が入れ替わります。
// @swap演算子の戻り値はvoidです。[出典: Twitter / 10000favs]

関数呼び出し

FunctionCall ::= Function ( (&? Expr( , &? Expr)*)? ) |
                 Expr . FunctionCall  』 // (2).ToStr()などのメソッド呼び出し
// 関数定義時( Func )の引数の型の前に & を付け、関数呼び出し時も & を付けると、参照渡しになります。
// 例：func f(x : &int) に対し、do f(&i) [出典: Twitter / kuina_tesso]
// 数値型のメソッドを利用する際、例えば5.ToStr() ではなく、
// (5).ToStr() または 5 .ToStr() とする必要があります。[出典: Twitter / 10000favs]

Function ::= FunctionName |
             Variable | ConstantName 』 // 関数型で定義された変数・定数

型

Type ::= int | float | char | bool | // 符号付整数型・浮動小数点型・文字型・論理型
         sbyte8 | sbyte16 | sbyte32 | sbyte64 | // 符号付整数型
          byte8 |  byte16 |  byte32 |  byte64 | // 符号なし整数型
         list  < Type > | // リスト構造
         stack < Type > | // スタック構造
         queue < Type > | // キュー構造
         dict  < Type , Type > | // 辞書型
         func  < ( (&? Type( , &? Type)*)? ) (: Type)? > | // 関数型
         EnumName | ClassName | // 列挙体・クラス
         AliasName | // 別名
         [ ] Type 』 // 動的配列
// complex/money/ratio型 はしばらく仕様から外れることになりました。[詳細: Twitter / kuina_tesso]
// Kuinでは型が厳密に扱われます。暗黙の型変換はできません。
// 配列は、var array:[]int :: @new [5]int のように宣言でき、 array :~ @new [2]int のように要素数を増やせます。

値指定

Value ::= Expr( @to Expr)?( , Value)* 』
// switch文・try文で用いる値の指定方法です。
// Exprの型はそれぞれの文で制限されています。

コメント

Comment ::= { ( [^"'{}]+ | String | Char | Comment )* } 』
// 複数行コメント可能です。一行コメントに特化したコメント記法はありません。
// Kuinでは、コメントがネストできます。
// また、コメント内部に文字列リテラル、文字リテラルを含み得ます。
// 例えば、{ " }" {  } }は全体がコメント扱いされます。
